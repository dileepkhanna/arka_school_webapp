# SmartEduConnect — Capacitor Plan

## Overview

Wrap the existing React/Vite PWA into a native Android & iOS app using Capacitor. The web app code stays **100% unchanged** — Capacitor just shells it into a native WebView with access to native device APIs.

**Key advantage over Flutter:** No rewrite. Same React codebase, same API calls, same UI — just packaged as a native app.

---

## 1. How It Works

```
React App (Vite build)
        ↓
  Capacitor Core
        ↓
  Native WebView
  ┌─────────────┐
  │  Android    │  → APK / AAB (Play Store)
  └─────────────┘
  ┌─────────────┐
  │    iOS      │  → IPA (App Store)
  └─────────────┘
```

The `npm run build` output (`dist/`) is copied into the native project. Capacitor bridges JS ↔ native for device APIs (camera, push, storage, etc.).

## 5. Native API Helpers

```ts
// src/lib/capacitor.ts
import { Capacitor } from '@capacitor/core';
import { PushNotifications } from '@capacitor/push-notifications';
import { Camera, CameraResultType, CameraSource } from '@capacitor/camera';
import { Filesystem, Directory } from '@capacitor/filesystem';
import { Share } from '@capacitor/share';
import { Network } from '@capacitor/network';

export const isNative = Capacitor.isNativePlatform();

// Push Notifications (replaces VAPID web push when native)
export async function registerPushNotifications() {
  if (!isNative) return; // fall back to existing VAPID push on web

  const permission = await PushNotifications.requestPermissions();
  if (permission.receive !== 'granted') return;

  await PushNotifications.register();

  PushNotifications.addListener('registration', (token) => {
    // Send FCM token to backend: POST /push-subscriptions
    fetch('/api/push-subscriptions', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ token: token.value, platform: Capacitor.getPlatform() }),
    });
  });

  PushNotifications.addListener('pushNotificationReceived', (notification) => {
    console.log('Push received:', notification);
  });

  PushNotifications.addListener('pushNotificationActionPerformed', (action) => {
    // Navigate based on notification data
    const url = action.notification.data?.url;
    if (url) window.location.href = url;
  });
}

// Camera / Image Picker (replaces <input type="file">)
export async function pickImage(): Promise<string | null> {
  if (!isNative) return null; // web uses native file input

  const image = await Camera.getPhoto({
    quality: 80,
    allowEditing: false,
    resultType: CameraResultType.DataUrl,
    source: CameraSource.Prompt, // asks user: Camera or Gallery
  });

  return image.dataUrl ?? null;
}

// File Download (replaces browser download)
export async function saveAndShareFile(
  content: string,
  filename: string,
  mimeType: string
) {
  if (!isNative) return; // web uses existing blob download

  const result = await Filesystem.writeFile({
    path: filename,
    data: content,
    directory: Directory.Cache,
    encoding: 'utf8' as any,
  });

  await Share.share({
    title: filename,
    url: result.uri,
    dialogTitle: 'Save or share file',
  });
}

// Network status
export async function getNetworkStatus() {
  const status = await Network.getStatus();
  return status.connected;
}
```

## 6. Integrating with Existing Code

### 6.1 Push Notifications

The existing `usePushNotifications.tsx` hook uses VAPID web push. Wrap it to use native push when running in Capacitor:

```ts
// src/hooks/usePushNotifications.tsx — add at top of hook
import { isNative, registerPushNotifications } from '@/lib/capacitor';

useEffect(() => {
  if (isNative) {
    registerPushNotifications(); // use FCM via Capacitor
  } else {
    // existing VAPID web push logic stays here
  }
}, []);
```

### 6.2 Image Upload (Holiday Calendar, Gallery, etc.)

Wherever `<input type="file" accept="image/*">` is used, add native camera support:

```tsx
// Example: HolidayCalendar image upload
import { isNative, pickImage } from '@/lib/capacitor';

const handleImageSelect = async () => {
  if (isNative) {
    const dataUrl = await pickImage();
    if (dataUrl) setSelectedImage(dataUrl);
  }
  // else: existing file input handles it
};
```

### 6.3 CSV/PDF Downloads

```ts
// src/lib/downloadUtils.ts — wrap existing download logic
import { isNative, saveAndShareFile } from '@/lib/capacitor';

export async function downloadCSV(csvContent: string, filename: string) {
  if (isNative) {
    await saveAndShareFile(csvContent, `${filename}.csv`, 'text/csv');
  } else {
    // existing blob download logic
    const blob = new Blob([csvContent], { type: 'text/csv' });
    const url = URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = `${filename}.csv`;
    a.click();
  }
}
```

### 6.4 Offline Detection

```ts
// src/hooks/useNetworkStatus.ts
import { isNative, getNetworkStatus } from '@/lib/capacitor';
import { Network } from '@capacitor/network';

export function useNetworkStatus() {
  const [isOnline, setIsOnline] = useState(true);

  useEffect(() => {
    if (isNative) {
      Network.addListener('networkStatusChange', (status) => {
        setIsOnline(status.connected);
      });
    } else {
      window.addEventListener('online', () => setIsOnline(true));
      window.addEventListener('offline', () => setIsOnline(false));
    }
  }, []);

  return isOnline;
}
```

## 7. Backend Changes Required

### 7.1 FCM Token Storage

The existing push subscription endpoint stores VAPID web push subscriptions. Add FCM token support:

```php
// backend — push_subscriptions table needs a `fcm_token` column
// Migration: add_fcm_token_to_push_subscriptions_table.php
Schema::table('push_subscriptions', function (Blueprint $table) {
    $table->string('fcm_token')->nullable();
    $table->string('platform')->default('web'); // 'web' | 'android' | 'ios'
});
```

```php
// PushNotificationController.php — handle FCM token registration
public function store(Request $request) {
    if ($request->has('token')) {
        // FCM token from Capacitor native app
        PushSubscription::updateOrCreate(
            ['user_id' => auth()->id(), 'platform' => $request->platform],
            ['fcm_token' => $request->token]
        );
    } else {
        // existing VAPID web push subscription
        // ... existing logic
    }
}
```

### 7.2 Send Push via FCM

```php
// NotificationService.php — add FCM sending alongside VAPID
use Kreait\Firebase\Messaging\CloudMessage;

public function sendToUser(User $user, string $title, string $body, array $data = []) {
    // Send VAPID web push (existing)
    $this->sendVapidPush($user, $title, $body, $data);

    // Send FCM for native apps
    $fcmTokens = $user->pushSubscriptions()
        ->whereNotNull('fcm_token')
        ->pluck('fcm_token');

    foreach ($fcmTokens as $token) {
        $message = CloudMessage::withTarget('token', $token)
            ->withNotification(['title' => $title, 'body' => $body])
            ->withData($data);
        $this->messaging->send($message);
    }
}
```

## 8. Build & Run

### Initial Setup

```bash
# 1. Build the web app
npm run build

# 2. Initialize Capacitor (first time only)
npx cap init SmartEduConnect com.smarteduconnect.app --web-dir dist

# 3. Add platforms
npx cap add android
npx cap add ios

# 4. Sync web build to native projects
npx cap sync
```

### Development Workflow

```bash
# After any code change:
npm run build && npx cap sync

# Open in Android Studio
npx cap open android

# Open in Xcode
npx cap open ios

# Live reload during development (optional)
# Set server.url in capacitor.config.ts to your local IP
# npx cap run android --livereload --external
```

### Production Build

```bash
# Android APK/AAB
npm run build
npx cap sync android
# Then in Android Studio: Build → Generate Signed Bundle/APK

# iOS IPA
npm run build
npx cap sync ios
# Then in Xcode: Product → Archive
```

## 13. Comparison: Capacitor vs Flutter vs PWA

| Aspect | Capacitor | Flutter | PWA |
|--------|-----------|---------|-----|
| Code reuse | 100% (same React) | 0% (full rewrite) | 100% |
| Dev effort | Low (days) | High (months) | Minimal |
| Native feel | Good (WebView) | Excellent (native widgets) | Moderate |
| Push notifications | FCM (native) | FCM (native) | VAPID (web) |
| App Store | ✅ Yes | ✅ Yes | ❌ No |
| Play Store | ✅ Yes | ✅ Yes | ✅ TWA |
| Offline support | Via PWA cache | Custom | Via SW cache |
| File system access | ✅ Full | ✅ Full | ⚠️ Limited |
| Camera access | ✅ Native | ✅ Native | ⚠️ Limited |
| Bundle size | ~5-15MB + web assets | ~15-30MB | N/A |

**Recommendation:** Capacitor is the fastest path to a Play Store / App Store app since the entire React codebase is reused as-is.
